阿里巴巴为何不推荐使用线程池？

  1. newFixedThreadPool，singleThreadExecutor：采用了无界队列，主要问题是堆积的请求处理队列可能会耗费非常大的内存，甚至OOM
  2. cachedThreadPool 的maximumPoolSize为Integer.MAX_VALUE，可能会创建非常多的线程，深圳OOM

阿里巴巴推荐什么？自定义线程，避免Integer.MAX_VALUE及无界队列
